# Data Model

All tables are in the `public` schema in Supabase. All timestamps are UTC (`timestamptz`). All primary keys are UUID generated by `gen_random_uuid()`.

---

## Tables

|---|---|---|
| `id` | uuid | PK |
| `name` | text | Business name |

| `email` | text | |

| `timezone` | text | IANA timezone string |

### `availability_windows`
Weekly recurring availability for a staff member. Override rows (with `override_date` set) take precedence over recurring rows.

| Column | Type | Notes |
|---|---|---|
| `id` | uuid | PK |
| `business_id` | uuid | FK → businesses.id |
| `staff_id` | uuid | FK → staff.id |
| `day_of_week` | integer | 0=Sun … 6=Sat (used when override_date is null) |
| `start_time` | time | Local time, no timezone |
| `end_time` | time | Local time, no timezone |
| `override_date` | date | Set for date-specific overrides; null for weekly recurring |
| `is_blocked` | boolean | true = staff is unavailable this window |

---

## Relationships

```
businesses
  ├── staff              (business_id)
  ├── services           (business_id)
  ├── clients            (business_id)
  ├── bookings           (business_id)
  ├── availability_windows (business_id)
  ├── reminders          (business_id)
  ├── call_sessions      (business_id)
  ├── client_insights    (business_id)
  ├── communication_logs (business_id)
  ├── deposits           (business_id)
  ├── onboarding_state   (business_id)
  └── waitlist_entries   (business_id)

clients
  ├── bookings           (client_id)
  ├── reminders          (via bookings)
  ├── call_sessions      (client_id)
  ├── client_insights    (client_id)
  ├── communication_logs (client_id)
  └── waitlist_entries   (client_id)

bookings
  ├── reminders          (booking_id)
  ├── deposits           (booking_id)
  └── communication_logs (booking_id)
```
